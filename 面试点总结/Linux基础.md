# Linux系统的管理组成
## Linux内核
### 定义
- 内核指的是一个提供<font color="#ff0000">硬件抽象</font>层、磁盘及文件系统<font color="#ff0000">控制</font>、多任务等功能的系统软件
- 内核是一个<font color="#00b0f0">操作系统</font>的<font color="#ff0000">核心</font>, 是操作系统<font color="#ff0000">最基本的部分</font>. 它负责<font color="#ff0000">管理</font>系统的进程、内存、设备驱动程序、文件和网络系统等, 决定着系统的性能和稳定性
- 硬件抽象为应用软件和硬件提供了一套简洁、统一的<font color="#ff0000">接口</font>, 使程序设计更为简单
### 功能
- **进程管理**
	- 创建、调度和终止进程/线程
	- 实现多进程并发执行
	- 提供进程间通信机制 (IPC) 机制, 如管道、信号、共享内存、消息队列等
	- 管理进程优先级和调度策略
- **内存管理**
	- 虚拟内存管理
	- 物理内存分配与回收
	- 页面置换、交换机制
	- 内存保护 : 防止进程越界访问其他进程或内核空间
- **文件管理**
	- 支持多种文件系统
	- 提供统一的VFS (虚拟文件系统) 抽象层, 使不同文件系统对用户透明
	- 管理文件的创建、读写、删除、权限控制等操作
	- 缓存机制提升 I/O 性能
- **设备驱动**
	- 提供硬件设备的抽象接口, 使应用程序无需直接操作硬件
	- 支持字符设备、块设备和网络设备
	- 驱动可编译进内核或以模块形式动态加载
- **网络管理**
	- 实现完整的 TCP/IP 协议栈
	- 支持套接字编译接口
	- 提供防火墙、流量控制等高级功能
	- 支持无线网络、虚拟网络设备等

# make
- `make`是GNU提供的一种<font color="#ff0000">半自动化的工程管理器</font>
	- <font color="#00b0f0">半自动化</font>是指在使用工程管理器之前需要<font color="#ff0000">人工编写</font>程序的<font color="#ff0000">编译规则</font>, 所有编译规则都保存在makefile文件中
	- 全自动化的工程管理器会在编译程序前自动生成makefile文件

| 无 Make        | 有 Make               |
| ------------- | -------------------- |
| 手动输入冗长命令      | 一键构建 (`make`)        |
| 全量编译，效率低下     | **增量编译**，极大提升效率      |
| 依赖关系靠人脑记忆，易出错 | 依赖关系明确定义在 Makefile 中 |
| 构建过程不一致，易出错   | 构建过程标准化，可重复          |

# make和makefile的区别
- `make`是一个<font color="#ff0000">命令行工具</font>, 用于自动化构建和管理项目. 根据`makefile`的规则决定如何编译、链接或执行
- `makefile`是一个<font color="#ff0000">文本文件</font>, 包含了一些列规则, 告诉`make`如何构建目标

# cmake
`cmake`是一个跨平台的开源构建系统生成器, 用于管理和自动化软件项目的编译过程. 它本身不直接编译代码, 而是根据`CMkaeLists.txt`生成适用于不同平台和编译器的构建文件, 然后由对应的底层构建工具来实现执行编译

# Linux系统中文件的类型及其标志
- **-  (普通文件)** : 存储普通数据
- **d (目录文件)** : 存放目录项
- **l  (符号链接)** : 指向另一个文件的快捷方式, 包含目标文件的路径
- **c  (字符设备文件)** : 提供无缓冲的串行输入输出
- **b (块设备文件)** : 提供有缓冲的随机访问, 按数据块操作
- **p  (管道文件)** : 用于进程间通信的管道
- **s  (套接字文件)** : 用于网络通信和进程间通信
